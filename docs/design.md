# 自動化ラン制ゲーム 設計書 v0.2.1


- 検証指標を全面差し替え

## 変更履歴（v0.2. → v0.2.1）

## 1. コンセプト

- 1ラン12分（暫定）で「時間内にどれだけ稼げるか」を競う自動化ゲーム。
- レイアウト作業なし。悩みの中心は投資判断とボトルネック解消。
- アニメーションは情報伝達の一次手段。詰まりは数字より先に絵で分かる。

## 2. 設計目標（受け入れ条件）

「理論上の最適解をなくす」は目標にしない。以下を満たすことを目標とする。

- G-1: どの周回補正でも同一の固定戦略が首位にならない
- G-2: 新設・強化・貯金それぞれが最適になるゲーム状態が存在する
- G-3: 残り時間・バッファ状態・台数によって投資順位が実際に逆転する
- G-4: 上位2戦略のスコア差が極端にならない
- G-5: 連続無操作時間が15〜20秒を超えない

## 3. ゲーム構造

### 3.1 生産ラインと状態モデル

[採取器群] → bufferA → [加工器群] → bufferB → [出荷口]

- 状態変数: 所持金, スコア, bufferA量, bufferB量, 各工程の台数/強化Lv
- 更新は固定時間刻み（仮: 100ms/tick）。描画フレームに依存させない
- 各tick:
  - 採取器は bufferA に空きがある分だけ生産（満杯なら停止=blocked）
  - 加工器は bufferA から取り出せる分だけ処理（空なら starved）、
    bufferB に空きがある分だけ出力（満杯なら blocked）
  - 出荷口は bufferB から取り出せる分だけ出荷し、単価×個数を収入化
- バッファ容量は有限（仮: 各20。simulatorで掃引して決定）
- min()式は設計説明用の定常近似としてのみ使用。実計算はバッファ入出力
- ラン終了時の残バッファはスコアに換算しない

### 状態判定規則
- 各工程の状態は完全停止のみを starved / blocked とする。
  部分充足（空き容量や在庫が需要より少ないが0ではない）は running 扱い。
- 「詰まりかけ」の表現は状態ではなくバッファ充填率（A-2）が担う。
- game-core は各工程の utilization（実効処理量 ÷ 需要量、0〜1）を
  state に保持し、renderer / simulator が補助指標として参照できる。


### 3.2 系統（工程単位）

- 系統は3種（題材はTBD-1。仮: 金属/樹脂/ガラス）
- 系統は装置単位ではなく**工程単位**。採取・加工・出荷の各工程に1系統を設定
- 同一工程内の装置は同系統に限定。多数決・主力判定は使わない
- 工程の系統はラン中に変更可能か → TBD-2（初期案: 変更不可。ラン開始時に3工程分を選択）

### 3.3 投資

投資対象は5スロット: 採取 / bufferA(搬送) / 加工 / bufferB(搬送) / 出荷

| 選択 | 効果 | 性格 |
|---|---|---|
| 新設 | 装置工程: 台数+1 / 搬送: 容量+α | 即効・線形 |
| 強化 | 1台あたり処理量+10%（仮）。効果はN秒かけて線形に立ち上がる | 遅効・非線形 |
| 高額装置 | 1種のみ（3.5）。貯金して狙う転換点 | 跳躍 |

- 強化の対象は装置3工程（採取・加工・出荷）のみ。
  搬送（bufferA/B）への投資は新設（容量+α）のみとする。

- 強化立ち上がりN: 60秒と120秒の2案をプロトタイプで比較（30秒は12分ランでも弱い）
- コスト式: 新設 = base × 1.15^n（仮）。強化価格は台数連動を含めて設計する
  （新設価格だけが指数上昇すると後半は強化固定解になるため）
- 価格・効果はラン開始時から全開示

### 3.4 シナジー（効果軸分離）

- 統一（3工程が同系統）: 投資効率系。新設費用-20% かつ 強化立ち上がり半減（仮）
- 混成（3工程が全て異系統）: 収入系。出荷単価+40%（仮）
- 最終収益倍率同士の直接比較を不能にすることが目的。
  拮抗はsimulatorで検証し、どちらかの採用率が70%を超えたら数値を調整する
- 倍率上限規則（合計×1.5以内・乗算重ね掛け禁止）はシナジー同士に適用する。
  シナジー×周回補正の複合（例: mixed×compactBuffers）は上限の対象外とするが、
  複合ケースのバランスは検証指標（§6）で必ず確認する。
- 特定ペアはv0.2スコープ外（保留リスト）

### 3.5 高額装置（1種のみ）

- 要件: 購入後に投資の価値順位が変わる「転換点」であること
- 仮案「二次加工器」: bufferBの製品をさらに高級品化して単価×3。
  ただし専用の処理能力を持ち、新たなボトルネック段として追加される
  → 購入前は採取/加工強化が正解だった盤面が、購入後は二次加工と出荷の
    増強が正解に変わる
- 効果の具体はTBD-3。ただし「単なる大倍率」は禁止
- 二次加工器は精錬品バッファ（容量 仮:10。configで調整可）を持つ。
  満杯時は精錬を停止する（bufferBからの取り出しも行わない）。
- 精錬品バッファは有限バッファ原則（§3.1）に従い、rendererから
  充填率を参照できること。
- 出荷は精錬品を通常品より優先する（単価が高いため。プレイヤー選択なし）。
- ラン終了時の精錬品残はスコアに換算しない（§3.7と同じ扱い）。

### 3.6 周回補正（3種・関係変更型）

- 補正は「特定系統の大幅割引」のような答えの指定を禁止し、
  投資間の関係を変えるものに限定する。仮案:
  - 強化の立ち上がり時間 1/2
  - バッファ容量-30%、代わりに出荷単価+30%
  - 新設価格上昇率 1.15 → 1.10
- ラン開始時に1つ抽選
- compactBuffersの容量倍率は初期容量にのみ適用する。
  ラン中の増設（bufferA/B新設 +α）は等倍とする。
  （補正ラン中は増設の相対価値が上がる。これは意図した挙動）

### 3.7 スコアとラン終了

- スコア = 出荷による累計収入。投資してもスコアは減らない
- 所持金 = 投資用残高（スコアとは別管理）
- 終了時所持金はスコアに加算しない
- 貯金の失敗 = 貯金中に見送った投資の機会損失として自然に表現される

## 4. アニメーション要件

受け入れ条件: 数値表示を読まずに以下が分かること。

- A-1: 資源が粒として流れる
  - 粒は表示専用。game-coreのバッファ量・流量から生成（1粒=N個の代表表示）
  - 粒オブジェクト再利用、最大表示粒数を設定
- A-2: 詰まりが渋滞として見える（最重要・最難）
  - 表示は在庫数でなく**バッファ充填率0〜100%**から変換する
    - 0〜50%: 粒数が増える
    - 50〜90%: 粒間隔が狭くなる
    - 90〜100%: 搬送路の圧縮/点滅表現
    - 100%: 上流装置の停止が明確に見える
- A-3: 装置状態のモーション切替
  - game-coreが状態を返す: running / starved / blocked / ramping
  - 描画側で状態を推測しない。表示切替に最低継続時間200〜500ms
- A-4: 購入と効果発生の演出分離
  - 購入時: 設置演出。ramping中: 充填ゲージ。完全効果到達: 完了演出
- A-5: 出荷演出の時間集約
  - 100〜500ms単位で出荷をまとめ「+125 / 12個出荷」形式で表示
  - 個別粒の出荷表現と金額表示の更新頻度を分離

マイルストーン1の対象はA-1〜A-3の最小表現。A-4/A-5は後続。

## 5. 技術方針

- HTML + Vanilla JS + Canvas
- 構成分離:
  - game-core.js: 固定時間刻みの状態更新。全定数・全計算式の唯一の正
  - renderer: requestAnimationFrameで描画。game-core状態の純粋な投影
  - simulator.js: game-coreのみを高速実行
  - particles: バッファ量・流量から生成する純表示（ゲーム計算に不関与）
- 収益・進行を描画フレーム数に依存させない
- セーブ: ランセーブとメタ進行を別キーでlocalStorage保存
- 体制: Codex=実装（範囲限定・設計判断禁止）、Claude=設計・レビュー、人間=採用判断

## 6. 検証指標（simulator.js）

- 補正別に各戦略の平均・中央値・下位分位スコア
- 同一固定戦略が全補正で首位になる割合（G-1）
- 新設/強化/貯金それぞれが最適になる状態の存在確認（G-2, G-3）
- 上位2戦略のスコア差（G-4）
- 有効な購入判断の発生間隔、連続無操作時間（G-5）
- 高額装置購入前後での投資優先順位の変化（3.5の転換点検証）
- バッファ容量の掃引（小さすぎ=頻繁停止 / 大きすぎ=在庫戦略復活 の境界確認）

## 7. プロトタイプ範囲（マイルストーン1）

含む: 1テーマ / 3系統 / 3工程+搬送2 / 有限バッファ / 新設・強化 /
高額装置1種 / 統一・混成 / 周回補正3種 / 12分ラン / A-1〜A-3最小表現

保留（基本ループ成立後に判断）: 特定ペア / メタ進行 / 注文・特売 /
高額装置の複数化 / 30分ラン化 / A-4・A-5

30分化の前提条件: 終盤まで投資順位が変わる / 転換点が最低2回成立 /
連続待ち時間15〜20秒以内 / 3択に死に択がない、を12分版で確認すること。
